## 1. ツール概要

### 動作環境

| 項目 | 要件 |
|------|------|
| OS | Windows 10 (20H2以降推奨) / Windows 11 |
| .NET ランタイム | .NET 8 Desktop Runtime（セルフコンテインド配布の場合は不要） |
| WebView2 Runtime | Microsoft Edge WebView2 Runtime（Windows 10 20H2以降・Windows 11は標準インストール済み） |
| ffmpeg.exe | 別途入手して配置が必要（詳細: `docs/ffmpeg_setup.md`） |
| ffprobe.exe | 任意。配置すると動画情報（長さ・解像度等）を画面表示できる |
| ネットワーク | 不要（完全オフライン動作） |
| 管理者権限 | 不要（実行は一般ユーザー権限で可） |

## 2. 配置手順

### 2-1. 配布パッケージの構成

```
MovieConverter/
├── MovieConverter.exe         ← アプリ本体
├── bin/
│   └── ffmpeg/
│       ├── ffmpeg.exe         ← 別途入手して配置（必須）
│       ├── ffprobe.exe        ← 任意。配置すると動画情報を表示
│       ├── avcodec-*.dll      ┐
│       ├── avformat-*.dll     │ 配布元ZIPのDLL一式を
│       ├── avutil-*.dll       │ ffmpeg.exe と同じフォルダに配置
│       ├── swresample-*.dll   │ (ffmpeg.exe 単体では動作しない
│       ├── swscale-*.dll      │  ビルドがある)
│       └── その他のDLL        ┘
├── Assets/
│   └── player.html            ← プレイヤー用HTML（必須）
├── docs/
│   ├── ffmpeg_setup.md
│   └── license_notice.md
└── manuals/
    └── user_manual.md
```

### 2-2. 配置手順

1. 配布パッケージを任意のフォルダに展開する
   - 例: `C:\Tools\MovieConverter\`
   - 全利用者が使う場合は共有フォルダへの配置を検討する（ただし出力先は各自の端末内を推奨）

2. `docs/ffmpeg_setup.md` を参照し、`ffmpeg.exe` および同梱DLL一式を `bin\ffmpeg\` フォルダに配置する
   - `ffmpeg.exe` 単体だけでは動作しないビルドがある
   - 配布元ZIPのフォルダ構成を崩さずにコピーすること

3. コマンドプロンプトで動作確認を行う（**配布前に必ず実施**）

   ```bat
   bin\ffmpeg\ffmpeg.exe -version
   ```

   バージョン情報が表示されれば配置は正常。表示されない場合はDLL不足の可能性がある。

4. 動作確認を行う（`docs/test_scenarios.md` の正常系テスト）

5. 利用者に `manuals/user_manual.md` を配布する

### 2-3. 初回確認チェックリスト

配布前・初回セットアップ後に以下をすべて確認すること。

- [ ] `MovieConverter.exe` をダブルクリックしてアプリが起動する
- [ ] 起動後、プレビュー領域に「MP4ファイルを選択すると...」と表示される（WebView2 初期化成功）
- [ ] コマンドプロンプトで `bin\ffmpeg\ffmpeg.exe -version` を実行してバージョン情報が表示される
- [ ] サンプル MP4 をファイル選択で読み込み、プレビューが表示される
- [ ] サンプル MP4 を **ドラッグ＆ドロップ**で読み込み、プレビューが表示される
- [ ] 「しない（高速カット）」で変換実行し、出力ファイルが作成される
- [ ] 「標準」（圧縮変換）で変換実行し、出力ファイルが作成される
- [ ] プレビューできないMP4を読み込み、事前変換のダイアログが表示される
- [ ] 事前変換を実行すると、元ファイルと同フォルダに `_preconverted_` 付きのファイルが作成される
- [ ] 事前変換完了後、変換後の動画が自動的にプレビューに読み込まれる
- [ ] 日本語フォルダ名（例: `D:\動画素材\`）のMP4が読み込める
- [ ] スペースを含むフォルダ名（例: `C:\My Videos\`）のMP4が読み込める
- [ ] 出力ファイルが入力ファイルと同じフォルダに作成されることを確認する

## 3. ビルド手順（ソースからのビルド）

開発環境が必要な場合の手順です。

**必要なもの:**
- .NET 8 SDK
- NuGetパッケージ（インターネット接続のある環境でのリストア）: `Microsoft.Web.WebView2`

**ビルドコマンド:**

```bash
cd src/MovieConverter
dotnet restore
dotnet build -c Release
```

**セルフコンテインド単一ファイル発行（推奨）:**

```bash
dotnet publish -c Release -r win-x64 --self-contained true -p:PublishSingleFile=true -o publish/
```

発行後は `publish/MovieConverter.exe` が単一の実行ファイルとして生成されます。
この場合、.NET 8 ランタイムのインストールが不要になります。

## 4. アップデート手順

1. 旧バージョンの実行ファイルをバックアップする
2. 新バージョンのファイルを配置する
3. `bin\ffmpeg\ffmpeg.exe` は既存のものを引き継ぐ（更新が不要な場合）
4. 更新後に動作確認を行う
5. 利用者に更新を周知する

## 8. 障害対応

### 8-1. アプリが起動しない

| 確認事項 | 対応 |
|----------|------|
| .NET 8 Desktop Runtime がインストールされているか | https://dotnet.microsoft.com からインストール（セルフコンテインド配布の場合は不要） |
| `MovieConverter.exe` が「ブロックされている」状態でないか | ファイルのプロパティ→「ブロックの解除」チェックを確認 |
| Windowsイベントログにエラーがあるか | イベントビューアの「Windowsログ→アプリケーション」を確認 |

### 8-2. 動画プレビューが表示されない / 再生できない

| 確認事項 | 対応 |
|----------|------|
| WebView2 Runtime がインストールされているか | コントロールパネル→プログラムで「Microsoft Edge WebView2 Runtime」を確認 |
| `Assets\player.html` が存在するか | 配布パッケージに含まれているか確認 |
| WebView2 のユーザーデータフォルダが破損していないか | `%TEMP%\MovieConverter_WebView2\` を削除して再起動する |
| プレビューできない MP4 がある | 事前変換ダイアログを試す。解決しない場合は DX 担当に相談 |

> **代替プレビューについて（v0.4.3以降）:**
> WebView2 Runtime が未インストールの端末では、アプリが自動的に代替プレビューモードで動作します。
> ffmpeg によるサムネイル表示でカット位置を確認でき、変換実行も可能です。
> 詳細はセクション 12「代替プレビューに関する管理者向け情報」を参照してください。
>
> （参考）v0.3.1 で WPF MediaElement による代替プレビューを実装しましたが、`UseWpf=true` による起動クラッシュが発生したため v0.3.2 で無効化しました。v0.4.3 では WPF を使わない方式（ffmpeg + WinForms PictureBox）で再実装しています。

#### 起動時例外ログ（v0.3.2）

アプリが起動できない場合、`{アプリフォルダ}\logs\startup_error_{timestamp}.log` に例外情報が記録されます。

| 項目 | 内容 |
|------|------|
| ログ場所 | `{MovieConverter.exe と同じフォルダ}\logs\` |
| ファイル名 | `startup_error_yyyyMMdd_HHmmss.log` |
| 記録内容 | 日時・バージョン・OS・例外種別・メッセージ・スタックトレース |

アプリが全く起動しない場合は、ログフォルダを確認してから Windows イベントビューアーも確認してください。

#### MP4/AVC(H.264) なのにプレビューできない場合（v0.1.2改訂）

MP4/AVC/H.264/AACの動画であっても、WebView2のプレビューで再生できない場合があります。
これはAVC非対応ではなく、プレビュー読み込み方式との相性問題です。

**v0.1.2の対応:** file-uri方式（主）→ virtual-host方式（フォールバック）の自動切り替えを実装済み。ログで確認できます。

**ログ確認:**
- `[プレイヤー] file-uri方式で読み込みます` → 主方式で試行中
- `[読み込み成功] 方式: file-uri` → file-uri方式で成功
- `[フォールバック] file-uri方式が失敗しました。virtual-host方式を試みます。` → フォールバック発動
- `[読み込み成功] 方式: virtual-host` → フォールバック方式で成功
- `[プレイヤー エラー] プレビューで再生できませんでした。` → 両方式とも失敗

**両方式とも失敗した場合:** 事前変換のダイアログが表示されます（v0.2.0以降）。「実行する」を選択すると自動的に事前変換を実行します。それでも再生できない場合は以下の切り分け手順を実施します。

**両方式とも失敗した場合の切り分け手順:**

**Step 1 — Microsoft Edge での直接再生確認**

対象MP4をEdgeのウィンドウにドラッグ＆ドロップして再生できるか確認する。

- Edge で再生できない → 動画ファイル自体に問題がある可能性が高い
- Edge で再生できる → 下記 Step 2 以降を試す

**Step 2 — 30秒サンプルで確認**

```bash
ffmpeg -ss 00:10:00 -t 00:00:30 -i input.mp4 -c copy sample_30sec.mp4
```

サンプルで再生できる場合 → 長さ・容量が原因の可能性あり

**Step 3 — faststart 化で確認**

```bash
ffmpeg -i input.mp4 -c copy -movflags +faststart output_faststart.mp4
```

faststart化後に再生できる場合 → moov atomの末尾配置が原因だった

**v0.2.0の対応:** プレビューに失敗した場合、事前変換のダイアログを案内します。利用者が同意した場合のみ事前変換を実行し、完了後に変換後ファイルを自動読み込みします。

**事前変換の仕様（管理者向け）:**

| 項目 | 内容 |
|------|------|
| 内部処理 | MP4を再生・読み込みしやすい構造に再配置する処理（moov atomを先頭に移動） |
| ffmpegオプション | `-c copy -movflags +faststart` |
| 再エンコード | なし |
| 画質劣化 | なし |
| 元ファイルへの影響 | なし（別ファイルを作成） |
| 出力ファイル名 | `{元ファイル名}_preconverted_{yyyyMMdd_HHmmss}.mp4` |
| 必要容量 | 元ファイルとほぼ同等の空き容量が必要 |

### 8-3. 変換に失敗する

変換失敗時、ダイアログに原因の目安が自動的に表示されます（v0.4.0以降）。

| 表示されるメッセージ | 原因 | 対応 |
|----------|------|------|
| DLL ファイルが不足している | ffmpeg のDLL不足 | DLL一式を `bin\ffmpeg\` に配置し直す |
| 書き込みが拒否された | 出力先の権限不足 | フォルダのアクセス権を確認する |
| ディスク容量が不足 | 空き容量不足 | 出力先ドライブの空き容量を確認する |
| 入力ファイルが破損 | ファイル破損・形式不一致 | 別ファイルで確認する |

その他の確認事項:

| 確認事項 | 対応 |
|----------|------|
| `bin\ffmpeg\ffmpeg.exe` が存在し実行可能か | コマンドプロンプトで `bin\ffmpeg\ffmpeg.exe -version` を実行して確認 |
| ffmpeg.exe がウイルス対策ソフトにブロックされていないか | 除外設定を検討する |

#### 終了コード `-1073741515` が表示される場合

`ffmpeg.exe` は見つかっているが、必要なDLLが不足しています。

- `bin\ffmpeg\` フォルダに配布元ZIPのDLL一式（`avcodec-*.dll`、`avformat-*.dll`、`avutil-*.dll`、`swresample-*.dll`、`swscale-*.dll` 等）を配置してください
- `bin\ffmpeg\ffmpeg.exe -version` を実行してバージョン情報が表示されることを確認してください
- 表示されない場合は配布元のZIPから再度DLLを取り出して配置してください
- ライセンス・配布元・同梱ファイルの扱いは管理者が確認してください

### 8-4. 変換に時間がかかる

圧縮変換（画質優先・標準・容量優先）は動画全体を再エンコードするため、長時間動画では時間がかかります。

| 状況 | 対応 |
|------|------|
| 高速化したい | 「しない（高速カット）」を試す（再エンコードなし・高速） |
| 圧縮変換が必要だが遅い | 解像度を 720p または 480p に下げると変換速度が向上する場合がある |
| 変換中の進捗を確認したい | 画面上部のプログレスバーと経過時間を確認する（v0.1.5以降） |

### 8-5. 高速カットで開始位置がずれる

高速カット（`-c copy`）はキーフレーム単位でカットするため、指定位置より数秒前にずれる場合があります。これは ffmpeg の仕様です。

| 状況 | 対応 |
|------|------|
| 数秒のずれが許容できる | そのまま使用可能 |
| 精度が必要 | 「標準」等の圧縮変換を選択する（再エンコードするためカット位置が正確になる） |

## 9. 一時ファイルの管理

| ファイル | 場所 | 管理方法 |
|----------|------|---------|
| WebView2 ユーザーデータ | `%TEMP%\MovieConverter_WebView2\` | アプリが自動管理。問題時は手動削除可 |
| 代替プレビュー一時ファイル | `%TEMP%\MovieConverter_preview\` | アプリ終了時に自動削除。WebView2 未インストール環境でのみ作成 |
| 変換途中ファイル | 入力ファイルと同フォルダ | キャンセル・失敗時はアプリが自動削除 |
| ログ保存ファイル | 入力ファイルと同フォルダ（未選択時は `{exeフォルダ}\logs\`） | 利用者が「ログを保存」ボタンで任意保存。自動では作成されない |
| 診断結果ファイル | `{exeフォルダ}\logs\diagnostic_{timestamp}.txt` | 利用者が「動作確認」→「診断結果を保存」で任意保存 |
| 起動時ログ | `{exeフォルダ}\logs\startup_latest.log` | 起動のたびに上書き更新。バージョン・OS・WebView2・ffmpeg情報を記録 |
| 起動時例外ログ | `{exeフォルダ}\logs\startup_error_{timestamp}.log` | 未処理例外発生時のみ作成。クラッシュ調査に使用 |

## 10. 問い合わせ対応: 動作確認（環境診断）の活用

利用者から「動作しない」「変換できない」等の問い合わせがあった場合、まず **「動作確認」** 機能を実行してもらうことで、原因の切り分けが容易になります。

### 手順

1. 利用者にアプリを起動してもらう
2. ログ欄上部の「**動作確認**」ボタンを押してもらう
3. 確認完了後に表示されるダイアログで「**診断結果を保存**」を押してもらう
4. 保存された `logs/diagnostic_{timestamp}.txt` を管理者に送ってもらう

### 診断結果の確認ポイント

| 表示 | 意味 | 対応 |
|------|------|------|
| `[ NG ]  ffmpeg.exe: 未配置` | ffmpeg.exe が配置されていない | `bin\ffmpeg\ffmpeg.exe` を配置する |
| `[ NG ]  ffmpeg 起動確認: DLL 不足` | DLL一式が揃っていない | 配布元ZIPのDLL一式を `bin\ffmpeg\` に配置する |
| `[注意]  WebView2 Runtime: 未インストール` | WebView2 Runtime がない | 代替プレビュー（ffmpeg サムネイル）が起動するため変換は可能。WebView2 をインストールすると動画再生プレビューが使用可能になる |
| `[ NG ]  出力先フォルダ: 書き込み不可` | 出力先に書き込み権限がない | フォルダのアクセス権を確認する |
| `[注意]  ffprobe.exe: 未配置` | ffprobe.exe が配置されていない | 動画情報表示が不要であれば対応不要 |

### startup_latest.log の活用

起動のたびに `logs/startup_latest.log` が更新されます。端末情報とツールの配置状況を一覧できるため、リモート対応時の環境確認に利用できます。

## 12. 代替プレビューに関する管理者向け情報（v0.4.3以降）

WebView2 Runtime が未インストールの端末では、アプリは自動的に代替プレビューモードで動作します。

**プレビュー方式の比較:**

| 方式 | 条件 | 動画再生 | カット位置確認 | 変換 |
|------|------|----------|--------------|------|
| 標準プレビュー | WebView2 インストール済み | 可（動画再生） | 再生しながら確認 | 可 |
| 代替プレビュー | WebView2 未インストール | 不可（サムネイルのみ） | ナビゲーションボタンで確認 | 可 |

**代替プレビューの詳細動作:**

| 状況 | 動作 |
|------|------|
| WebView2 未インストール + ffmpeg 配置済み | サムネイル表示でカット位置確認が可能。変換実行可能 |
| WebView2 未インストール + ffmpeg 未配置 | サムネイルは表示されないが、テキスト入力で位置指定して変換できる |
| WebView2 インストール済み | 標準プレビュー（動画再生）を使用。代替プレビューは表示されない |

**一時ファイルについて:**

代替プレビューのサムネイルは `%TEMP%\MovieConverter_preview\thumb.jpg` に保存されます。アプリ終了時に自動削除されます。異常終了した場合は手動で削除してください。

**WebView2 のインストール（必要な場合）:**

`docs/test_scenarios.md` セクション 8-2 を参照してください。WebView2 Runtime は Microsoft の公式サイトから取得するか、Windows Update 経由でインストールできます。閉域環境では管理者が個別配布する必要があります。
