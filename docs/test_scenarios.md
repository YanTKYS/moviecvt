## 1. 前提条件

- ffmpeg.exe が `bin\ffmpeg\ffmpeg.exe` に配置されていること
- WebView2 Runtime（Microsoft Edge WebView2 Runtime）がインストールされていること
- テスト用 MP4 ファイルが用意されていること（3〜5分程度を推奨）
- テスト用非MP4ファイルが用意されていること（例: test.avi, test.txt）

### 8-3. MP4/AVC プレビュー失敗時の切り分け手順

プレビューに失敗した場合、以下の手順で原因を切り分けてください。

#### ステップ 1: Microsoft Edge での直接再生確認

対象のMP4ファイルをMicrosoft Edgeで直接開いて再生できるか確認する。

- Edge で再生できない場合 → 動画ファイル自体に問題がある可能性が高い
- Edge で再生できる場合 → 本ツール（WebView2）の読み込み方式との相性問題の可能性あり

#### ステップ 2: 短いサンプルで確認

長時間・大容量動画の場合、30秒程度のサンプルを切り出して確認する。

```bash
ffmpeg -ss 00:10:00 -t 00:00:30 -i input.mp4 -c copy sample_30sec.mp4
```

- サンプルでは再生できる場合 → 動画の長さ・容量がWebView2プレビューの制約に引っかかっている可能性あり

#### ステップ 3: faststart 化で確認

MP4のメタデータ位置（moov atom）が末尾にある場合、先頭への移動（faststart）で改善する場合がある。

```bash
ffmpeg -i input.mp4 -c copy -movflags +faststart output_faststart.mp4
```

- faststart化後に再生できる場合 → moov atom 末尾配置がWebView2での読み込み失敗の原因だった
- faststart化後も再生できない場合 → 他の要因を確認する

#### 切り分け結果の記録

| 確認項目 | 結果 | 備考 |
|----------|------|------|
| Edge で直接再生できるか | - | |
| 30秒サンプルで再生できるか | - | |
| faststart化後に再生できるか | - | |
